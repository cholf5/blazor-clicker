# ADR 0004：范围边界 —— 只做「纯增长」版本

- **状态**：已采纳
- **日期**：2026-07-04
- **前置**：[ADR 0003](0003-late-game-and-polish.md)

---

## 背景

原版 Cookie Clicker 经过十几年迭代，在核心的「建筑 → 升级 → 成就」增长
循环之外，陆续叠加了大量后期系统：花园（Garden）、股市（Stock Market）、
万神殿（Pantheon）、魔法书（Grimoire）、奇美拉龙（Dragon）、Wrinklers /
Grandmapocalypse、以及各类季节活动。原版成就总数因此达到 622 个，而本
项目在补齐档位后为 ~334 个。

本 ADR 回答一个反复出现的问题：**这个差距要不要追平？** 结论是——不追。
本项目**刻意**只实现原版早期（约 2013 年）那个「纯指数增长玩具」的形态，
并主动拒绝绝大多数后期管理型系统。这份文档记录这个取向背后的推理，好让
未来任何人（包括维护者自己）再看到「成就只有 334 个」时，不会重新萌生
「要不要加 Garden 来凑数」的念头。

---

## 采纳的决策

### 1. 项目的灵魂是「单一清晰的进度轴」

**决策**：本 remake 的设计目标不是「复刻原版的全部内容」，而是复刻它
**早期那个纯粹的指数增长体验**：

> 玩家做的每个决定都指向同一个方向——数字变大。买建筑、买升级、看数量级
> 从 million 爬到 billion。没有岔路，没有维护成本，没有「你没在最优操作」
> 的焦虑。

**理由**：

- 早期 CC 的乐趣来自 **单一、清晰、可读的进度轴（single legible
  progression）**。增长本身**慢**，所以每一次数量级跨越都是一个「事件」。
- 这个体验是自洽且完整的。它不需要后期系统来「拯救」——恰恰相反，正是
  「没有别的事要操心」让它成为一个好的 idle 玩具。
- 本项目已经把这个灵魂做得相当完整：建筑、升级、成就（334 个，数据驱动）、
  金饼干、糖块、转生。这已经是一个「做完了」的游戏，不是半成品。

### 2. 主动拒绝「主动管理型」后期系统

**决策**：以下系统**不做**，且这是一个设计决策，不是待办事项：

| 系统 | 不做的原因 |
| --- | --- |
| Garden（花园） | 主动微操，要求定期回来照料、记机制、承担「没最优操作 = 亏」的心理负担——与 idle「放着就好」的核心诉求方向相反 |
| Stock Market（股市） | 同上，是电子表格式的主动管理 |
| Pantheon（万神殿）/ Grimoire（魔法书） | 同上，且依赖糖块消耗系统（ADR 0003 §2 已推迟） |
| 季节活动 | 需要真实日期驱动 + 活动切换机制，工程量巨大而与核心增长无关 |

**理由**：

- 这些系统把一个 **idle 玩具**，改造成了一个需要**上班打卡**的管理游戏。
  社区里「CC 后期变成 spreadsheet simulator」是常见吐槽，指的就是这个
  性质转变。
- 它们的乐趣曲线是**长期养成型**的，与本项目「点击 → 数字变大」的**即时
  反馈**是两种不同的游戏，强行并存会稀释而非增强核心体验。
- 它们本是 Orteil 为**已经玩腻了纯增长的极后期老玩家**准备的新维度。本
  项目**不服务那个游玩阶段**，因此没有理由背上它们的复杂度。

### 3. Wrinklers / Grandmapocalypse：不做，或仅保留纯正面的可选形态

**决策**：默认**不做**。Wrinklers / Grandmapocalypse 引入了「要不要主动
触发一个负面事件来换取更高收益」的风险决策，这会把干净的增长轴**污染**成
一个需要权衡的系统。

**理由**：

- 与 §1 的「单一清晰进度轴」直接冲突。
- 若未来确实想要，只允许做**纯正面、纯可选**的变体（例如一个不带负面
  惩罚、玩家可完全忽略的收集小彩蛋），绝不做强制或带惩罚的版本。

**代价**：放弃约 15–35 个相关成就。可接受——见 §5。

### 4. Dragon（奇美拉龙）：可选，非承诺

**决策**：Dragon 是唯一**可能**未来考虑的后期系统，但**不是承诺**。

**理由**：

- 它是**线性养成**（喂养 → 升级），不强制主动微操，对「放着就好」的
  伤害最低，能套进现有「数据驱动 + 纯 domain 逻辑」的架构。
- 但即便要做，也应在核心体验之外作为独立、可忽略的支线，且优先级低于
  任何能强化纯增长轴的改动。

### 5. 不以「成就数量追平原版」为目标

**决策**：成就总数**不追求**与原版（622）持平。当前 ~334 个是**结果**，
不是需要修正的**缺口**。

**理由**：

- 原版那多出来的成就，绝大部分挂在 §2–§3 拒绝的系统上。既然不做那些
  系统，对应成就自然不存在——这是取向的**必然结果**，不是遗漏。
- 追平数字会倒逼我们去做自己都不信的系统，本末倒置。
- 若要继续扩充成就，正确方向是**在纯增长轴上加**，而不是靠新系统凑数：
  - 更细的 CPS / baked 里程碑档位；
  - 「单次转生内达成 X」这类**约束型**成就（只需新计数器，即 ADR 0003
    之后的 B 类思路）；
  - 组合条件（「每种建筑至少 1 个」、「只靠点击烤到 X」）。
- 这些和现有架构完全同构（一行 `record` + 可选一个计数器），且不背叛
  游戏的灵魂。

---

## 主动没做的事

- **Garden / Stock Market / Pantheon / Grimoire**：见 §2，永久不做。
- **季节活动**：见 §2，永久不做。
- **Wrinklers / Grandmapocalypse（带惩罚的默认形态）**：见 §3。
- **以数量追平原版成就**：见 §5。

---

## 已知的限制 / 债务

- 本 ADR 是一个**取向声明**，不是永久封印。若未来对这个游戏的定位发生
  根本改变（例如决定认真服务极后期玩家），应新开一份 ADR 显式推翻本文，
  而不是悄悄地加系统。
- §3 / §4 给 Wrinklers 的纯正面变体和 Dragon 留了一扇窄门；开门的前提是
  它们**不损害** §1 的单一进度轴，否则同样落入「不做」。
